# Как работает Kanban-оркестрация в Hermes Agent

Разбираем архитектуру мультиагентной координации через доску Kanban — от dispatch до handoff.

## Что такое Kanban-оркестрация

В Hermes Agent каждый агент — это автономный воркер, который получает задачи из общей Kanban-доски. Доска реализована как очередь с приоритетами, где каждая задача имеет статус: `backlog`, `in_progress`, `review`, `done`.

## Dispatch задач

Когда поступает новый запрос, диспетчер анализирует его и разбивает на подзадачи. Каждая подзадача отправляется в соответствующую колонку доски. Агенты подписываются на колонки, которые соответствуют их специализации.

## Handoff и контекст

При передаче задачи между агентами (handoff) сохраняется полный контекст:
- История диалога
- Промежуточные результаты
- Метрики выполнения
- Зависимости от других задач

## Синхронизация

Агенты синхронизируются через общую SQLite-базу. Каждый агент периодически пуллит доску на предмет новых задач, а после завершения — обновляет статус.

## Итог

Kanban-оркестрация позволяет эффективно распределять нагрузку между агентами, не терять контекст при передаче задач и легко масштабировать систему добавлением новых воркеров.
